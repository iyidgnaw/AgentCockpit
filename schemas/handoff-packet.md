# Handoff Packet — Agent Handoff Protocol

The standard format for transferring context from one agent to another.
Generated by `create_handoff_packet`, consumed by `delegate_to_agent`.

## Format

~~~markdown
## Objective
[One sentence: what the next agent should accomplish]

## Current State
Agent: {runtime} | State: {state} | Repo: {repo} | Branch: {branch}
Task hint: {task_hint}

## Evidence
### Screen (last 40 lines)
{screen_tail from source agent}

### Git Changes
{git diff --stat output, capped at 60 lines}

## Constraints
{list of constraints, or "None specified"}

## Requested Action
{one of: review | continue | test | debug | summarize}

## Completion Criteria
Signal completion by returning to an idle prompt state.
~~~

## Requested Action types

| Action | Meaning |
|---|---|
| `review` | Read and critique only — do not modify code |
| `continue` | Pick up implementation where the source agent stopped |
| `test` | Write or run tests for the current changes |
| `debug` | Investigate a failure or error and report findings |
| `summarize` | Summarize current state without taking action |

## Example

~~~markdown
## Objective
Review the payment retry implementation for correctness and edge cases.

## Current State
Agent: claude-code | State: idle | Repo: /Users/me/myapp | Branch: feature/payment-retry
Task hint: implement the retry backoff

## Evidence
### Screen (last 40 lines)
❯ implement the retry backoff
[implementation output...]
❯

### Git Changes
 src/payment/RetryService.ts | 45 +++++++++++++++++++++++-
 tests/payment/retry.test.ts | 23 +++++++++++
 2 files changed, 68 insertions(+)

## Constraints
None specified

## Requested Action
review

## Completion Criteria
Signal completion by returning to an idle prompt state.
~~~
